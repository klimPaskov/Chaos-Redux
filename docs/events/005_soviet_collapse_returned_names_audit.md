# Event 005 Returned Names Restoration Audit

Audit date: 2026-05-21

This audit covers the ancient and medieval restoration requirement from the final clean merged Part 6 specification. The implementation now has two layers: dynamic host-country Returned Names decisions for existing eligible countries, and standalone high-chaos ancient restoration tags for the Khazar, Sogdian, Khwarazmian, and Alan claim lines. The standalone tags are seed packages with country/history files, flags, localisation, focused spawn/setup logic, event notices, event-log stages, League interaction, and a shared restoration focus tree with tag-gated branch content.

## Remaining Deviation

The Returned Names system now satisfies the standalone-tag surface at seed-package level: each claim line has a tag, country/history files, flags, localisation, focus content, release/spawn logic, report event, and event-log integration. The focus tree now has 32 focuses and forces each standalone tag through a four-focus identity branch before the common symbolic or expansionist endgame. The focus icons now use stable per-focus `ancient_*.dds` paths with themed source reuse, and the four standalone tags have tag-specific sourced portrait DDS files. The remaining deviations are final-art and depth related: the new flags still use placeholder reuse; the four tags still share one tree framework instead of fully bespoke full trees; and live validation has not been performed in game.
