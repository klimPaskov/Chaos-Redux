# Event 006: Independence Wave

## Current documentation correction

Current source-of-truth map: `docs/plans/006_independence_wave_plans/source_of_truth_map.md`.

As of the 2026-06-05 user correction, Kuban (`KUB`) and Altai (`ALT`) are not part of the currently accepted Event 006 expansion direction. Earlier documentation and handoffs that framed them as implemented Event 006 high-chaos package carriers are superseded for source-of-truth purposes and should be treated as historical tranche notes, not current requested scope.

The currently accepted new-country lane is niche Independence Wave releases in underrepresented regions, especially Africa and South America. `ASN` Asante, `KBN` Kanem-Bornu, `DFR` Darfur, `ZUL` Zulu, `PLM` Palmares, `AYM` Aymara Highland Congress, and `MAP` Mapuche Land Congress use unique imagegen-regenerated flags. The shared `independence_wave_liberation_provisional_tree` remains generic; country identity is carried through decisions, spirits, route flags, claim targets, event details, and localisation instead of country-specific stacked focus blocks.

`MAP` Mapuche Land Congress is a separate accepted local-polity package, not part of the generic niche lane and not a Chile cosmetic fallback. It uses a real custom tag, custom country/history/localisation setup, unique upright flags, Event 006 package decisions, and the same shared Liberation Provisional Tree as other Event 006 releases.

The current wrap-up target is technical playability and urgent functional closure, not implementing every optional package, route variant, portrait, animation, or spreadsheet detail. Report and news event image GFX files now exist, and current Event 006 report/news texture references resolve; those assets are no longer a completion blocker. Remaining report/news work is limited to optional future variants if later routes need more visual specificity.

## Current implementation

Independence Wave is now routed as an Event 006 origin system instead of a generic release button. The active base pass compiles inactive candidates through one `every_possible_country` scan, validates each release against a live host, reserves a host-owned survival state before each release attempt, skips candidates that would consume every host-owned state, and stops releasing once the current chaos-tier target is reached. The event recipient is not a responsible actor and is not treated as a special host. A host can enter the resolver through war and instability pressure, surrender pressure, or a peripheral-colonial pressure gate where it owns and controls a non-capital candidate-cored state that is not one of the host's own cores. The target is now rolled inside the configured tier band: baseline waves aim for 3-5 releases, Evo I for 4-6, Evo II for 5-7, Evo III for 6-9, Evo IV for 8-12, and Evo V for 10-16, with host survival and valid candidates still allowed to reduce the actual count.

The current protected-state pass prefers the host capital when the candidate does not core it. If the capital cannot be reserved, it uses a strongest-state proxy in owned non-candidate-cored territory, preferring factories, dockyards, supply nodes, urban states, infrastructure, host cores, and then any remaining host-owned state. The protected state is stored through a temporary global event target that is cleared before and after each release attempt so it cannot leak between candidate checks. Successful release attempts also mark the protected state with `independence_wave_host_survival_reserved`, which blocks later Event 006 candidate releases or Border Commission claims from consuming it. Before `release = TAG`, the resolver now also picks a candidate-cored anchor state, temporarily masks the candidate's other host-owned cores, releases the country from that anchor, and restores the masked cores afterward as later claim and package-proof territory. Verified package anchors are tried before generic anchors: OGB prefers Kazan then Cheboksary, ASY prefers Mosul, DNZ prefers Danzig, UGA prefers Uganda, SOK prefers Sokoto, BUK prefers Bukhara, KHI prefers Khiva, IRQ prefers Baghdad `291`, DON prefers Rostov `218`, BAR prefers Barotseland, DAH prefers Dahomey, MIS prefers Nicaragua, ITZ prefers Belize, GAR prefers Misiones/Rio Grande/Chaco seats, CHR prefers the Charrua assembly seats, KUR can start from a controlled Kurdish mountain district, KBK starts from Kabardino-Balkaria/Nalchik `827`, NMB starts from Khomas `541`, BOT starts from Bechuanaland `542`, PAL starts from Palestine `454`, GHA starts from Ghana `274`, ERI starts from Eritrea `550`, DFR starts from South Darfur `887` while North Darfur `767` remains a petition/proof target, ZUL starts from Natal `719`, and MAP starts from Araucania `950` with Aysen `949` and Rio Negro `512` as later survey/recovery territory. Countries created through this path receive `independence_wave_reduced_territory_start` and extra claim ambition, while host survival remains mandatory.

The resolver clears the per-run candidate pool, the current-wave release ledger, the release counter, and the 16 current-wave country display slots before each wave. Candidate gates use Event 006-owned origin, existence, package, state, and host-survival checks; they do not read Soviet Collapse runtime flags, so Event 005 state cannot suppress an otherwise valid Event 006 wave. `global.independence_wave_released_countries` and `global.independence_wave_compact_members` are persistent ledgers used by targeted compact decisions, so repeatable waves must not wipe them while existing release flags remain on countries.

From Gathering Storm onward, the resolver prioritizes a niche generic-candidate lane before bespoke high-chaos packages. The lane uses inactive vanilla tags with existing state cores and flag assets, including Katanga, Biafra, Cabo Verde, Rif, Rapa Nui, Rio Grande, Welsh Argentina, and, at later tiers, Harar, Tigray, Afar, Benishangul-Gumuz, Gambela, Sidamo, Oromo, and Qemant when they are inactive and host-safe. At Chaos Tier and Totalen Chaos release-scale milestones, the lane can also seed custom high-chaos generic releases for Asante, Kanem-Bornu, Palmares, and the Aymara Highland Congress through verified non-capital anchor states. These countries load `independence_wave_liberation_provisional_tree`; African identity work is implemented through the Asante Stool Court, Kanem-Bornu Caravan Court, Acacia Frontier Pact, Lion Crown, Gorilla Chair, and African Dossier Congress decisions rather than bespoke focus stacks.

One-state releases are marked as limited-territory starts and receive an emergency border survey immediately, so their Border Commission can pursue surveyed-district recovery or first file adjacent parish claims instead of leaving them locked to the release anchor. Package-family scoring can mark dormant, city/port, railway, protectorate, historical-return, local-polity, and strange candidates for later overlays. The railway family now has live release scoring, Event 006 dossier tracking, and a first Formation Ledger route for high-infrastructure releases: the state assembles a junction committee, secures bridge guards, and can proclaim a Timetable Authority after proving hub control, legitimacy, and militia strength. Hosts receive an aftermath flag, host anger, a capital ministry mission, and the Independence Dossier Board. The board is a decision-category surface for the host crisis, exposing open dossier count, host anger, and foreign attention while giving the host concrete responses: recognition, observers, dossier talks, archive evacuation, local autonomy offers, courier arrests, loyalist councils, and capital-ministry continuity. Successful releases are also stored in the former host's `independence_wave_host_released_countries` ledger, so targeted Dossier Board actions only point at the countries that broke away from that host. Decision categories are hidden when empty so players only see boards that currently have playable work. Routine actions use short cadences with moderate political-power costs, while military, patronage, border, and congress actions use meaningful resource costs that fit the action: manpower for local brigades, command power for coercive orders and courier staff, equipment for rifle shipments and arms pools, army experience for adviser missions, coalition cohesion for congress tradeoffs, legitimacy for dependency breaks, and stability or foreign attention where the choice creates public risk. Timed missions state their fail and success outcomes in localisation: failure discredits the route or removes proof, while success records integration proof and keeps the route available for later settlement work.

The core boards now have a first decision-category scripted GUI value layer. `common/scripted_guis/006_independence_wave_scripted_guis.txt` mounts display containers from `interface/006_independence_wave_scripted_gui.gui` onto the Dossier Board, New States Congress, Patron Ledger, Formation Ledger, and Border Commission. The buttons, costs, tooltips, AI behavior, and targeted actions still live in the decision files; this layer is only the readable board surface.

The New States Congress now has concrete cooperation actions beyond attendance. `independence_wave_sign_mutual_guarantee` is a targeted congress decision between two independent Event 006 releases that have sent delegates; it creates reciprocal guarantees, raises coalition cohesion and legitimacy on both sides, adds foreign attention, and records the guarantee network for achievement tracking. `independence_wave_send_congress_volunteer_cadre` sends command staff, manpower, and infantry equipment from one congress member to another vulnerable release, raising target militia strength and coalition cohesion while recording the corridor. `independence_wave_open_congress_arms_pool` spends infantry and support equipment to open a shared reserve, raising militia strength, coalition cohesion, and legitimacy. These actions have AI weights tied to route identity, war state, anti-puppet proof, compact membership, and current militia strength.

The Great Partition Week super-event is wired to high-chaos mass-wave proof. The Rump That Endures super-event is wired to the host-survival resolver. Before each release, the host stores its pre-wave controlled-state count through `independence_wave_prepare_host_wave_tracking`; after release, `independence_wave_mark_host_aftermath` can fire slot `58`, `The Rump That Endures`, only when a chaos tier IV or V wave has reduced a host that began above the one-state floor to one or two controlled states after at least four counted release losses. The gate requires the host to remain alive, keeps the world-end bypass exclusion, and turns the absolute Event 006 rule into presentation proof: the host is humiliated and reduced, not deleted.

The League War super-event and achievement proof run from scoped war lifecycle hooks. `common/on_actions/006_independence_wave_on_actions.txt` watches `on_war_relation_added`, `on_capitulation`, `on_peaceconference_ended`, `on_puppet`, and `on_annex`. The revolutionary committee route gives radical releases an alternative to observer, officer, national, or patron alignment. The Patron Ledger is the first anti-patron management layer. It opens from the patron cabinet route, port-ledger/balanced-sponsor focuses, anti-puppet congress proof, or rising patron leverage. It lets a release accept advisers or an arms corridor for survival, balance rival patrons, expose a broker, reject dependency clauses, or convert emergency loans into public treaty debt. Every action uses a shared cooldown, updates `independence_wave_patron_leverage`, shows its route-specific cost in a compact icon/value tooltip, spends a route-appropriate resource such as army experience, command power, coalition cohesion, legitimacy, or corridor equipment, and has AI weights tied to route, war state, and anti-puppet proof.

The shared focus tree now gives that ledger a dedicated anti-patron recovery spine. `independence_wave_expose_broker_files` opens public broker files under high patron pressure, `independence_wave_audit_dependency_clauses` ties loan and cabinet clauses to Congress scrutiny, and `independence_wave_charter_without_masters` records a public anti-patron route state. The branch uses the existing Patron Ledger decisions instead of creating a second sponsor system. The tree layout keeps the five political mandates, crisis branch, shared support desks, Congress spine, Formation Ledger, and strange branch in separate columns; cross-route locks that do not need visible path lines use availability gates instead of drawing repeated OR-prerequisite fans.

The shared focus tree also has a cross-route crisis recovery spine for releases under war pressure, collapsing stability, failed compact work, or patron cabinet-seat demands. `independence_wave_summon_emergency_session` records an emergency public sitting, `independence_wave_form_street_defense_committees` turns the crisis into organized street defense, and `independence_wave_reopen_cabinet_authority` restores a signable cabinet chain and reconnects the state to recognition and Congress delegation tools. The branch uses existing pressure, militia, legitimacy, patron leverage, and AI posture variables rather than adding a separate crisis board.

Major and regional outside powers now have a separate `independence_wave_patronage_recognition_category` so patronage is not only breakaway self-management. Eligible non-Event-006 powers can target independent Event 006 releases to recognize a committee, ship rifles and support crates, offer a military mission, demand cabinet seats once leverage is already present, or sabotage a rival patron. These decisions use political power, equipment, army experience, and command power costs; they update the target's legitimacy, militia strength, foreign attention, patron leverage, recognition state, and patron-accepted achievement tracking without creating puppets, factions, country files, or Event 005 links.
