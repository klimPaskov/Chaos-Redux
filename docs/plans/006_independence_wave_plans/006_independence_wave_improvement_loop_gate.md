# Event 006 Independence Wave Improvement Loop Gate

The improvement loop should deepen Event 006 only while there is still meaningful design depth to add.

## Expansion pass is useful when

- a country package exists only as a tag
- a formation route has no reveal, proof, cost, or post-formation play
- a decision category is a passive store
- a scripted GUI only decorates the system
- a focus route is a reward ladder
- AI has no route logic
- assets do not reflect route state
- historical-return content uses old names without institutions, symbols, politics, or limits
- the event has no replayable consequences after a wave resolves

## Closure handoff is correct when

- accepted specs and accepted plans are implemented or intentionally queued
- no known simplifications or fallbacks remain
- country packages have identity, forces, routes, AI, localisation, and assets
- formation routes have map proof, reveal logic, integration, and AI safety
- GUI buttons have costs, tooltips, helpers, AI equivalents, and cleanup
- animations have source frames, static fallbacks, manifests, and gfx handoffs
- decisions have varied costs, failure states, dynamic text, and cleanup
- focus routes have payoffs, tradeoffs, AI, and visible consequences
- super-events mark real thresholds and have source research requirements
- event logs, event details, docs, catalog, localisation, assets, and validation align

## Closure handoff shape

A closure handoff should be short. It should not invent new route families.

It should say:

```text
No broad expansion is recommended. The system is now clean enough to finish because more mechanics would add bloat rather than gameplay value.
```

Then it should list only final tasks such as:

- run completion audit
- rerun localisation audit
- check asset handoffs
- check spreadsheet alignment
- validate host survival scenarios
- validate formation decisions
- validate GUI button requirements and AI equivalents
- validate no unresolved accepted plans remain

The main agent still owns the final completion claim.
